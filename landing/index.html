What it does

prsage runs as a GitHub App. When a pull request opens, it pulls the diff, sends it to an LLM tuned for code review, and posts inline comments on the lines worth flagging. Bugs, security holes, missing tests, real readability issues. No nitpicks.

The bot only comments. Humans still merge.

How it works

  1. 1

    PR opens

    You push a branch and open a pull request like normal.

  2. 2

    prsage reads the diff

    The webhook lands, signature gets verified, and the diff is pulled via the GitHub API.

  3. 3

    LLM review pass

    The diff goes to Groq's Llama 3.3 70B with a tuned reviewer prompt. Output is structured JSON.

  4. 4

    Inline comments posted

    Each comment is verified to land on a line that was actually added in the diff before being posted.

Stack

Python · FastAPI · SQLAlchemy · PostgreSQL · Groq (Llama 3.3 70B) · GitHub Apps · Railway · Vercel
